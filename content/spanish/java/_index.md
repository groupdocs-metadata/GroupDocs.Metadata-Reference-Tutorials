## Cómo cargar document java usando GroupDocs.Metadata
Cargar un documento es el primer paso antes de poder leer o editar su metadata. La biblioteca abstrae el manejo de archivos, permitiéndote trabajar con streams, rutas de archivo o URLs. Una vez que el documento está cargado, puedes consultar cualquier propiedad de metadata compatible.

*Pasos típicos:*
1. Crea una instancia de `Metadata` que apunte a tu archivo.  
2. Llama al cargador apropiado (p.ej., `PdfMetadata`, `ImageMetadata`).  
3. Accede al `RootPackage` para explorar o modificar propiedades.

Este flujo simple te permite **load document java** rápidamente, sin importar el tipo de archivo.

## Preguntas Frecuentes

**P: ¿Puedo cargar un PDF protegido con contraseña y aún así leer su metadata?**  
R: Sí. Proporciona la contraseña al inicializar el cargador, y la metadata será accesible.

**P: ¿La biblioteca soporta la lectura de metadata de archivos ZIP encriptados?**  
R: Puede leer metadata estándar de ZIP; para archivos encriptados debes proporcionar la contraseña de descifrado.

**P: ¿Cómo maneja GroupDocs.Metadata archivos grandes?**  
R: La biblioteca transmite datos, por lo que el consumo de memoria se mantiene bajo incluso para documentos de varios gigabytes.

**P: ¿Es posible eliminar toda la metadata de un archivo en un solo paso?**  
R: Sí, el método `clean()` elimina todas las propiedades de metadata compatibles mientras preserva el contenido original.

**P: ¿Qué versiones de Java son compatibles?**  
R: Java 8 y versiones posteriores son totalmente compatibles.
